# CEO Business School — 기능 로드맵

> 상태: 초안 (스택 확정 대기) · 작성일 2026-07-20
> 현재 사이트: 정적 브로슈어 11페이지, 동작 기능 0

## 0. 원칙
- **의존성 순서를 지킨다.** LMS·멤버십·구독결제는 전부 `인증 + DB`가 전제.
- **리드(자문/문의) 폼은 로그인 없이 가능** → 가장 빨리 성과가 보임.
- **관리자(Admin) 화면은 각 단계마다 최소한만 병행** 제작 (데이터 넣고 볼 수 있어야 함).
- **큰 빌드 전 관련 Next.js 문서 확인** (이 레포는 수정된 Next — AGENTS.md 규칙).

## 1. 기술 스택 (확정 대상)

| 영역 | 1순위 추천 | 대안 | 메모 |
|---|---|---|---|
| DB | **Supabase (Postgres) — 확정** | — | LMS는 파일저장·인증 동시 필요 → 통합형 유리 |

## 2. 데이터 모델 (초안)
- `users` — 회원 (auth 연동, 이름/회사/직함/연락처)
- `advisory_requests` — 자문 신청 (이름/연락처/회사/문의내용/상태)
- `contacts` — 일반 문의
- `courses` — 강의 (제목/설명/가격/영상ID/공개여부)
- `enrollments` — 수강 (user × course, 진도, 수료여부)
- `orders` — 주문 (금액/상태, 결제수단 무관)
- `payments` — 결제 시도/결과 (provider: 'toss' | 'token' | ..., 외부키, 상태, 금액)
  - provider 무관 인터페이스. 원화(토스)·자체 토큰(코인) 모두 이 아래로 꽂음
- `memberships` — CEO클럽 멤버십 (등급, 구독상태, 갱신일)
- `posts` — 인사이트/출판/연구 글 (선택: CMS화 전까지)

## 3. 단계별 계획

### Phase 0 — 기반 셋업 *(스택 확정 직후)*
- Supabase 프로젝트 생성, env 연결, 클라이언트 셋업
- DB 스키마 초안 마이그레이션
- 산출물: 로컬에서 DB 읽기/쓰기 동작

### Phase 1 — 리드 기능 *(빠른 성과, 로그인 불필요)*
- **자문 신청 폼**: `/advisory` 버튼 → 신청 폼 → `advisory_requests` 저장
- **문의 폼**: `/contact` → `contacts` 저장
- 관리자 알림(이메일) + 최소 관리자 목록 화면
- 산출물: 실제 신청/문의가 DB에 쌓이고 알림 옴

### Phase 2 — 인증/계정
- 회원가입·로그인 (이메일 + 카카오), 세션
- 로그인 버튼 실동작, 마이페이지 골격
- 산출물: 로그인 후 상태 유지, 내 정보 화면

### Phase 3 — 결제 기반
- **결제 어댑터 계층** 먼저 설계 (`PaymentProvider` 인터페이스, `orders`/`payments` 모델)
- 첫 provider 구현: 토스(원화) 또는 자체 토큰 중 그때 확정
- 자체 암호화폐 결제는 별도 provider로 추가 (지갑/컨트랙트 연동은 후속)
- 산출물: 결제 플로우가 provider 무관하게 동작 → 주문 기록 생성

### Phase 4 — 온라인 교육(LMS)
- 강의 목록 → 상세 → 수강신청 → 결제 → 영상 재생 → 진도/수료
- 관리자: 강의 등록/수정
- 산출물: 강의 구매 후 수강 가능

### Phase 5 — CEO클럽 멤버십 + 구독결제
- 정기결제(빌링), 등급, 회원 전용 콘텐츠 접근 제어
- 산출물: 멤버십 가입/갱신, 전용 콘텐츠 잠금

### Phase 6 — 확장
- 오프라인 교육 신청/일정·정원
- 콘텐츠 CMS화 (인사이트/출판/연구)
- AI경영도구 (별도 기획)

## 4. 확정 현황

- [x] 결제: **추상화 어댑터로 설계**, 하위 provider(토스/자체토큰)는 Phase 3에서 확정
- [ ] Phase 1(리드 폼) 먼저 착수 여부

## 5. 후속 확인 필요 (자체 암호화폐 결제)
- 토큰 종류: 자체 발행 코인? 기존 체인(이더리움/폴리곤 등) 위 토큰? 자체 체인?
- 지갑 연결 방식 (메타마스크 등) vs 내부 잔액 원장(off-chain) 방식
- 원화 병행 여부 (코인 전용 vs 원화+코인 선택)
- → Phase 3 진입 전 별도 기획
